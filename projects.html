Phishing Email Server & Exploit

Overview:

I conducted a controlled penetration testing simulation to evaluate user awareness and infrastructure vulnerabilities. This included developing a phishing email server that delivered crafted messages to simulated users, testing their response behaviors and susceptibility to social engineering tactics.

In parallel, I exploited a misconfigured Windows Server Certificate Authority (CA) to decrypt user certificates, revealing critical weaknesses in certificate lifecycle management and access control. The project emphasized ethical testing practices and secure exploit execution within a sandboxed environment.

Complex Challenges Faced:

  • Safely deploying phishing simulations without triggering real-world security incidents
  • Exploiting CA vulnerabilities while preserving system integrity
  • Balancing realism with ethical boundaries in exploit design
  • Troubleshooting environment setup across multiple virtual machines
  • Communicating technical findings in a way that emphasized education over blam

Ethical Considerations and Actions Taken:

  • Ensured all testing was conducted in a controlled, non-production environment
  • Maintained strict boundaries around exploit scope and data access
  • Focused on user education and awareness rather than punitive outcomes
  • Documented all procedures to support transparency and reproducibility

Outcomes:

  • Demonstrated key attack vectors and infrastructure weaknesses
  • Strengthened understanding of phishing tactics and certificate exploitation
  • Reinforced the importance of user training in cybersecurity defense
  • Produced actionable insights for improving certificate management and awareness programs

Smart Card Authentication

Overview:

I designed and deployed a secure authentication system using smart cards to enhance enterprise login security. This involved setting up a Certificate Authority (CA), issuing digital certificates, and configuring multi-factor authentication (MFA) protocols across client machines. The project emphasized Public Key Infrastructure (PKI) principles and enterprise-grade identity management.

The system was built to simulate a real-world environment where users authenticate using smart cards and PINs, ensuring strong access control and minimizing credential-based attacks.

Complex Challenges Faced:

  • Managing certificate lifecycles, including renewal and revocation processes
  • Ensuring compatibility between smart card readers and diverse client machines
  • Troubleshooting authentication failures and certificate trust issues
  • Balancing usability with strict security requirements
  • Documenting setup procedures for future scalability and training

Ethical Considerations and Actions Taken:

  • Ensured all certificate issuance followed secure identity verification practices
  • Maintained transparency in system limitations and user onboarding procedures
  • Prioritized user privacy and secure handling of authentication data
  • Created clear documentation to support ethical deployment and maintenance

Outcomes:

  • Successfully deployed a smart card-based authentication system with MFA
  • Strengthened understanding of PKI, certificate management, and secure login protocols
  • Demonstrated the effectiveness of hardware-based authentication in enterprise environments
  • Produced reusable documentation for future implementations and training
